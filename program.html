Programming Team

Programming Team Overview

The software for the AUV will be split into several sections. Two of these sections will be run on the Raspberry Pi microcontrollers. The remaining four sections will be run on the Arduino Uno microcontrollers. Several of each type of microcontroller are being used because one microcontroller would not have enough I/O ports and processing power to do all required tasks. The programs executed on the Arduinos will take sensor input, do a few simple calculations, and then output the data to one of the Raspberry Pi’s. This code will be written in the Arduino programming language and will heavily rely on third party libraries written by the manufacturers of the sensors that are being connected to it. The programs executed on the Raspberry Pi’s will check for changes in input from the Arduinos, run image processing and other algorithms that analyze any input, and then output any necessary adjustments to the motors. These tasks will be contained inside of an infinite loop so that it can continually make real-time changes to the AUV. This program will be written in Python and will be executed using the Python 3 interpreter that is pre-installed on the Raspberry Pi. This division of tasks between the Arduino and Raspberry Pi microcontrollers takes advantage of the Raspberry Pi’s superior processing power as well as the Arduino’s greater sensor software libraries and better I/O capabilities.
